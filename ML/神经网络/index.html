    神经网络¶

    神经网络本质上是一个万能函数逼近器：给它大量输入-输出样本，它能自动"学"出一个函数来拟合这种映射关系。

    举个例子——给你一堆房子的面积、卧室数、位置，要你预测房价。你可以手动写一个线性公式 房价 = w1×面积 + w2×卧室数 + w3×位置 + b，但神经网络做的事情差不多，只是它能学更复杂的非线性关系。

    核心思想就三步：（1）前向算结果 →（2）跟答案比差距 →（3）根据差距调参数，反复循环直到足够好。

    神经元¶

    神经元是神经网络的最小计算单元，模拟生物神经元"接收信号→汇总→决定是否激活"的过程。

    数学定义¶

    一个神经元接收 \(n\) 个输入 \(x_1, x_2, ..., x_n\)，每个输入有一个权重 \(w_i\)，加上一个偏置 \(b\)，经过激活函数 \(f\) 输出：

    \[y = f\left(\sum_{i=1}^{n} w_i x_i + b\right)\]

    直观理解¶

    • 权重 \(w_i\)：这个输入有多重要（正=促进，负=抑制）
    • 偏置 \(b\)：激活门槛的高低调整——偏置越大，即使输入信号较小也容易激活；偏置越小，需要更大的输入才能激活
    • 激活函数 \(f\)：把加权和映射到一个合理范围（比如 0~1 之间，像概率）

    举例¶

    假设一个神经元判断"今天适不适合出门"：

    • \(x_1\) = 温度 25°C（归一化到 0~1，25°C → 0.6），\(w_1 = 0.8\)
    • \(x_2\) = 降雨概率 0.8（80%），\(w_2 = -0.6\)
    • \(b = 0.3\)

    加权和 = \(0.6 \times 0.8 + 0.8 \times (-0.6) + 0.3 = 0.48 - 0.48 + 0.3 = 0.3\)

    再用 Sigmoid 激活：\(S(0.3) = \frac{1}{1+e^{-0.3}} \approx 0.574\)，偏向"适合出门"但没那么确定——因为温度适中但降雨概率高，两者互相抵消后偏置起了决定作用。

    神经网络结构¶

    多个神经元按层组织，前一层的输出作为后一层的输入，形成"多层感知机"。

    三层结构¶

    • 输入层：接收原始数据（不计算，只是占位），神经元数 = 特征数
    • 隐藏层：核心计算层，可以有多个，每层提取不同层次的特征
    • 输出层：给出最终结果，神经元数 = 要预测的值的个数

    全连接意味着：当前层的每个神经元都和上一层的所有神经元相连。

    为什么需要多层？¶

    单层只能表达线性关系。堆叠多层 + 非线性激活函数 = 能表达任意复杂函数。

    类比：一层 = 只能画直线分割数据；两层 = 能画折线；三层以上 ≈ 能画任意形状的边界。

    正向传播¶

    从输入 → 隐藏层 → 输出，逐层计算，最终得到预测值的过程。

    矩阵形式¶

    设第 \(l\) 层的权重矩阵为 \(W^{[l]}\)，偏置向量为 \(b^{[l]}\)，激活函数为 \(f^{[l]}\)：

    \[\begin{aligned} Z^{[l]} &= W^{[l]} A^{[l-1]} + b^{[l]} \\ A^{[l]} &= f^{[l]}(Z^{[l]}) \end{aligned}\]

    其中 \(A^{[0]}\) 即输入 \(X\)，\(A^{[L]}\) 即最终输出 \(\hat{Y}\)。

    计算示例¶

    一个两输入、两隐藏神经元的简单网络：

    • 输入 \(X = [2, 3]^T\)
    • 隐藏层权重 \(W^{[1]} = \begin{bmatrix} 0.5 & 0.2 \\\ 0.1 & 0.8 \end{bmatrix}\)，偏置 \(b^{[1]} = [0.1, 0.1]^T\)
    • \(Z^{[1]} = W^{[1]} X + b^{[1]} = \begin{bmatrix} 0.5 \times 2 + 0.2 \times 3 + 0.1 \\\ 0.1 \times 2 + 0.8 \times 3 + 0.1 \end{bmatrix} = \begin{bmatrix} 1.7 \\\ 2.7 \end{bmatrix}\)
    • 经 Sigmoid：\(A^{[1]} = [S(1.7), S(2.7)]^T \approx [0.846, 0.937]^T\)

    然后 \(A^{[1]}\) 继续向后传播，直到输出层。

    反向传播¶

    正向传播给出预测值后，需要衡量"差了多少"，然后把这个误差反向传回去，指导每个参数该调多少。

    为什么叫"反向"？¶

    误差从输出层开始，利用链式法则逐层往回计算梯度，所以叫反向传播。

    损失函数¶

    衡量预测值和真实值差距的标尺：

    均方误差（MSE，回归任务）： $\(L = \frac{1}{2}(y_{pred} - y_{true})^2\)$

    标准 MSE 是 \(\frac{1}{n}\sum(y_{pred} - y_{true})^2\)。这里加了 \(\frac{1}{2}\) 系数，纯粹是为了求导后消掉平方项的 2，让梯度表达式更简洁——不影响优化方向。

    交叉熵（分类任务）： $\(L = -\sum_{i} y_i \log(\hat{y}_i)\)$

    参数更新¶

    算出每个参数的梯度 \(\frac{\partial L}{\partial w}\) 后，沿着梯度的反方向更新：

    \[w := w - \eta \cdot \frac{\partial L}{\partial w}\]

    其中 \(\eta\) 是学习率，控制每步走多大——太大容易跳过最优点（振荡），太小训练太慢。

    直观类比¶

    你想调出一个完美的吉他音： 1. 弹一下（正向传播） 2. 听听差了多少（算损失） 3. 判断琴弦该拧紧还是拧松（反向传播算梯度） 4. 稍微拧一点（参数更新） 5. 重复 1-4 直到音准

    链式法则¶

    反向传播的数学基础。对于复合函数 \(L = f(g(h(x)))\)：

    \[\frac{dL}{dx} = \frac{dL}{df} \cdot \frac{df}{dg} \cdot \frac{dg}{dh} \cdot \frac{dh}{dx}\]

    每一层都把自己的梯度传递给前一层，就像接力棒一样。

    数值举例：

    \(L = (3x + 2)^2\)，\(x=1\)

    正向：\(x=1 → 中间=5 → L=25\)

    反向： - \(\frac{\partial L}{\partial 中间} = 2 × 5 = 10\) - \(\frac{\partial 中间}{\partial x} = 3\) - \(\frac{\partial L}{\partial x} = 10 × 3 = 30\) ✓（直接求导：\(2(3x+2)·3\)，\(x=1\) 代入 = 30）

    神经网络训练优化¶

    问题：全量梯度下降太慢¶

    每次更新参数需要遍历全部训练数据（几万甚至几百万样本），计算一次梯度代价极大，而且会卡在局部细节里无法"跳过"。

    \[w := w - \eta \cdot \frac{1}{N}\sum_{i=1}^{N} \nabla L_i(w)\]

    当 \(N = 100\) 万时，更新一步就要算 100 万次梯度，不现实。

    Mini-Batch 梯度下降¶

    不每次用全量数据，而是每次随机抽取一小批（如 32、64、128 个样本）来估算梯度。这样更新快，且随机性有助于跳出局部陷阱。

    \[w := w - \eta \cdot \frac{1}{m}\sum_{i \in batch} \nabla L_i(w)\]

    三种梯度下降对比：

    类型 每步用多少样本 优缺点
    全量(Batch GD) 全部 \(N\) 方向最准，但太慢
    随机(SGD) 1 个 很快，但方向极不稳定
    Mini-Batch 32~256 个 平衡效率和稳定性

    Mini-Batch 的问题与对策¶

    震荡问题：

    Mini-Batch 的梯度只是真实梯度的估计，有噪声。batch 越小，估计越不准，优化路径越"蛇形走位"。

    • batch_size = 1：极致震荡，几乎随机游走
    • batch_size = 256：比较平稳，方向可靠

    局部极值：

    梯度下降到山谷（局部最低点）就停下来了，但可能旁边有更深的谷（全局最低点）。不过实践中，高维空间的"真正的局部极值"其实不多——大部分是鞍点。

    鞍点：

    导数为 0，但不是极值点（像马鞍：一个方向凸，一个方向凹）。在高维空间中鞍点远比局部极值常见，这是困扰梯度下降的主要问题——停在鞍点不动了。

    SGD 动量梯度下降¶

    给优化加"惯性"——如果之前一直在往某个方向走，那这次更新时保留一部分这个方向的速度。

    \[\begin{aligned} v_t &= \beta v_{t-1} + (1-\beta)\nabla L(w_t) \\ w_{t+1} &= w_t - \eta \cdot v_t \end{aligned}\]

    其中 \(\beta \approx 0.9\)，控制"惯性"大小。

    直观类比：就像从山上滚下来的球——越滚越快，遇到小坑直接冲过去，不会被卡住。

    动量法的效果¶

    • 在一致下降的方向上加速（如滚下斜坡）
    • 在来回震荡的方向上互相抵消（如峡谷两侧）
    • 更容易冲过鞍点

    没有动量时的下降路径：之字形前进，在横向震荡中浪费大量步数 有动量时的下降路径：横向震荡被平滑，纵向下降被加速，更快到达谷底
