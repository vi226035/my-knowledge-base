    深度学习常见疑惑解答¶

    把入门时最容易卡住的问题，用大白话一次性讲清楚。

    第一部分：神经网络的底层逻辑¶

    Q1：batch size 变大，模型参数会变多吗？¶

    不会。 参数数量只由网络结构决定，和每次喂多少数据无关。

    打个比方：神经网络是一台榨汁机（固定的刀片 + 电机）。一次丢 1 个苹果还是一个筐 64 个苹果，榨汁机本身没有任何变化。变化的是"一次处理多少"而已。

    数学上：[64, 28] × [28, 128] = [64, 128]。64 张图同时乘的是同一份 128 维的权重矩阵——矩阵乘法的并行特性让 batch 内所有样本共享参数。

    Q2：代码里写了四个一模一样的 nn.Linear(28,128)，为什么不是重复废工？¶

    因为它们是四个独立的"盲盒"——公式一样，但肚子里装的随机初始值完全不同。

    每次调用 nn.Linear()，PyTorch 在内存里开辟一块新地盘，塞入完全不同的随机数作为初始权重。经过训练后，四个盲盒各自学到不同的东西——一个变成遗忘门，一个变成输出门，各司其职。

    关键不是公式长什么样，而是权重值被训成了什么样。

    Q3：初始权重为什么不能全设为 0？¶

    必须打破对称性。全 0 = 所有神经元永远做一模一样的事。

    打个比方：请了 128 个专家，初始时全是白纸一张（权重为 0）。你扔一张猫图进去，算出来的误差对 128 个专家的"批评"完全一样（梯度相同）→ 他们每次接收同样的修正 → 训到最后 128 个专家全部一模一样——等于只请了一个人，白花了 127 份工资。

    给微小的随机初始值，让专家们起跑线不同，最终各自掌管不同的特征。

    第二部分：CNN 和 RNN 的分工¶

    Q4：CNN 能处理图像，为什么还要 RNN？¶

    CNN 是一次性快照，RNN 是连续剧。

    CNN RNN
    适合数据 独立图片 序列（文字、语音、股价）
    输入之间 互不相关 前后依赖
    核心机制 卷积核滑动 隐藏状态 \(h_t\) 沿时间传递

    RNN 每读一个词，就把理解（\(h_t\)）传给下一时刻——像击鼓传花，当前的理解建立在之前所有内容之上。

    第三部分：从 RNN 到 LSTM¶

    Q5：RNN 有什么致命缺点？¶

    长期失忆——梯度消失。

    RNN 每一时刻的记忆传递都要做矩阵连乘。如果乘数小于 1，连乘几十次后梯度趋近于 0——第一页的内容彻底"糊了"。

    打个比方：复印一份文件，每次复印是前一次的 90% 清晰度。复印 50 次后，第一页的字已完全看不清。RNN 实际只能记住约 10~20 步内的信息。

    Q6：LSTM 怎么解决失忆？（核心）¶

    LSTM 造了一条"加法高速公路"——细胞状态 \(C_t\)。

    RNN LSTM
    更新方式 \(h_t = \tanh(W x_t + U h_{t-1})\) 全量覆盖 \(C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t\) 选择性加法
    梯度传导 矩阵连乘 × 逐元素乘 + 加法 +
    远距离依赖 10~20 步 100+ 步

    符号速查： - \(C_{t-1}\)：旧的长期记忆 - \(f_t\)：遗忘门（0~1，哪些旧记忆该扔） - \(i_t\)：输入门（哪些新信息该写） - \(\tilde{C}_t\)：候选记忆（本次的新内容） - \(\odot\)：逐元素相乘

    因为是 加法（+）而不是矩阵连乘（×），梯度可以沿 \(C_t\) 无损地传回几十上百步之前。这就是 LSTM 能记住远距离依赖的根本原因。

    Q7：LSTM 复杂的公式到底在做什么？¶

    用记课堂笔记来理解每个组件：

    组件 符号 课堂比喻 作用
    细胞状态 \(C_t\) 主笔记本 记录所有重要考点（长期记忆）
    隐藏状态 \(h_t\) 临时脑子 当前这秒想着的事，传给下一时刻
    遗忘门 \(f_t\) 橡皮擦 擦掉笔记本上过时的旧内容
    候选状态 \(\tilde{C}_t\) 草稿本 根据新内容总结出的"本节课精华"
    输入门 \(i_t\) 荧光笔 判断草稿上有多少值得写入笔记本
    输出门 \(o_t\) 聚光灯 从笔记本挑出当前时刻最需要的内容

    四步流程：

    1. 擦旧：橡皮擦 \(f_t\) 擦掉该忘的 → \(f_t \odot C_{t-1}\)
    2. 写新：荧光笔 \(i_t\) 判断草稿精华值不值得写 → \(i_t \odot \tilde{C}_t\)
    3. 更新：旧的 + 新的 = 此刻笔记 → \(C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t\)
    4. 输出：聚光灯 \(o_t\) 照笔记本，提取当前需要的 → \(h_t = o_t \odot \tanh(C_t)\)

    第四部分：训练四步曲¶

    Q8：loss → zero_grad → backward → step 到底在干什么？¶

    这是模型从瞎蒙进化到精准的全部秘密。用厨师做菜来理解：

    loss = criterion(outputs, labels)  # ① 尝菜，给差评
optimizer.zero_grad()              # ② 撕掉昨天的记过本
loss.backward()                    # ③ 反向追责，精准定锅
optimizer.step()                   # ④ 执行处分，修改配方

    ① loss = ... — 顾客尝菜，给出差评

    算网络瞎猜和正确答案差多少。分数越高 = 错得越离谱。

    ② optimizer.zero_grad() — 先撕掉昨天的记过本

    PyTorch 的梯度默认累加！不清空的话这批失误会和上批混在一起。先清零，只算当前这批的账。

    ③ loss.backward() — 反向追责，精准定锅

    系统顺着流水线往回倒推（链式求导），算出每个参数"多调 1 单位能让差评减多少"——这叫梯度。

    比如：糖多 0.1 能大幅降差评 → 给糖标负梯度（该加糖了）；盐多 0.1 让差评暴增 → 给盐标正梯度（盐放多了）。

    执行完这行时参数还没被改——只是拿到了修改建议书。

    ④ optimizer.step() — 执行处分，修改配方

    真正修改权重的时刻：

    \[W_{new} = W_{old} - \eta \cdot \frac{\partial L}{\partial W}\]

    \(\eta\) 是学习率（比如 0.001），\(\frac{\partial L}{\partial W}\) 是梯度。一次改动极小，但几万个回合下来，瞎猜变成神准。

    第五部分：激活函数¶

    Q9：激活函数到底在干嘛？为什么不能直接去掉？¶

    来看一个具体例子。假设你要区分两种水果——苹果和橙子：

    • 苹果：直径 7cm，重 180g
    • 橙子：直径 7cm，重 200g

    如果只看直径，分不出来。如果只看重量，也分不太准。但如果你能同时看 \(\text{直径}^2 + \text{重量}^2\)、\(\text{直径} \times \text{重量}\)、\(\sin(\text{直径})\) 这样弯弯绕绕的组合，一下子就分开了。

    激活函数的作用就一句话：把输入"揉变形"，让简单的东西变复杂，从而能表达任意形状的分界线。

    没有激活函数会怎样？¶

    假设一个三层的网络：

    输入 x → 乘 W1 → 乘 W2 → 乘 W3 → 输出 y

    无论叠多少层，最终都等于：\(y = (W_3 \times W_2 \times W_1) \times x = W_{总} \times x\)

    就是一条直线！ 一百层的网络 = 一层。深度完全浪费了。

    打个比方：你叠了 100 张透明的直线图纸，透过它们看下面的曲线——曲线还是曲线，叠多少层直线都不可能变成曲线。

    加了激活函数后呢？¶

    输入 x → 乘 W1 → [ReLU 掰弯] → 乘 W2 → [ReLU 掰弯] → 乘 W3 → 输出 y

    每层之间加一个 ReLU（把负数全掐断），数据就不再是简单的直线变换了。一百层叠加 = 一百次掰弯 = 可以表达极其复杂的分界线。

    打个比方：用一根铁丝（直线），你只能做一根针。但有了"掰弯器"（激活函数），每次弯一下，弯 100 次之后——你能拧出一朵花。

    三大激活函数怎么选？¶

    先说最简单的理解方式：

    函数 你可以想象成… 什么时候用它
    ReLU 一把铡刀——正数放行，负数砍掉 CNN 卷积层、全连接层（95% 的情况）
    Sigmoid 一个阀门——输出永远在 0 到 1 之间 二分类输出、LSTM 的门控开关
    Tanh 一台压路机——输出永远在 -1 到 1 之间 RNN 隐藏状态、LSTM 候选记忆

    具体公式和特点：

    函数 公式 输出范围 为什么好 为什么有时候不好
    ReLU \(\max(0, x)\) \([0, \infty)\) 正区间梯度恒为 1，不会消失；计算最快 负数全部归零，"神经元死亡"
    Sigmoid \(\frac{1}{1+e^{-x}}\) \((0, 1)\) 天生适合做 0/1 开关 x 很大或很小时梯度趋近 0（梯度消失）
    Tanh \(\frac{e^x-e^{-x}}{e^x+e^{-x}}\) \((-1, 1)\) 0 中心对称，梯度比 Sigmoid 大 两端同样会梯度消失

    为什么 LSTM 门控用 Sigmoid，候选状态用 Tanh？¶

    这是一个新手常问的好问题——同样的 LSTM，不同位置选了不同的激活函数，不是拍脑袋定的。

    门控用 Sigmoid：因为门的本质是"开关"，需要输出 0~1： - 0 = 全关（遗忘门说"这段记忆全扔了"） - 1 = 全开（输入门说"这条信息全写进去"） - 0.5 = 半开（折中） - Sigmoid 天然输出 0~1，完美契合。

    候选状态用 Tanh：因为候选记忆是"新知识本身"，需要正和负： - 正数 = "这条特征支持这个结论" - 负数 = "这条特征反对这个结论" - 0 = "无关" - Tanh 输出 -1~1，以 0 为中心对称，完美契合。

    一句话：Sigmoid 管"要不要"，Tanh 管"是什么"。

    总结速查¶

    问题 一句话答案
    Batch size 影响参数数量？ 不影响，参数只由网络结构决定
    四个相同 Linear 是废工吗？ 不是，初始值不同，训出来各自不同
    权重能初始化为 0 吗？ 不能，打不破对称性，所有神经元永远一样
    CNN vs RNN？ CNN 处理独立图片，RNN 处理序列
    RNN 致命缺点？ 梯度消失，记不住远距离信息
    LSTM 怎么治的？ 加法更新 \(C_t\)，梯度可无损远传
    backward vs step？ backward 只是算建议，step 才真正改参数
    激活函数为什么要非线性？ 没有它网络就是一条直线，什么都分不开
    为什么门控用 Sigmoid？ 天然输出 0~1，完美模拟开关信号

    第六部分：反向传播和梯度下降的底层数学¶

    前面 Q8 讲了训练四步曲的流程，这里扎进数学底层，看清 backward() 和 step() 到底在算什么东西。

    Q10：反向传播到底在算什么？求导到底有什么用？¶

    用一个极简的例子——只一个参数 \(w\)，Loss 是 \(w\) 的平方：

    \[L(w) = w^2\]

    目标：找到让 \(L\) 最小的 \(w\)（显然答案是 \(w=0\)，但计算机不知道，要一步一步试）。

    第 1 步：前向传播（Forward）— 算当前有多差

    假设 \(w\) 当前值是 \(3\)，代入公式：

    \[L = 3^2 = 9\]

    误差是 9，有点大。接下来怎么调 \(w\) 才能让 9 变小？

    第 2 步：反向传播（loss.backward()）— 算”斜率”

    对公式求导：\(L = w^2\) 的导数是 \(2w\)。代入 \(w=3\)：

    \[\frac{dL}{dw} = 2 \times 3 = 6\]

    这就是 backward() 唯一做的事：算出此时此刻的倾斜度是 6。

    正数 6 意味着：如果你把 \(w\) 往正方向（变大）推，\(L\) 会跟着变大（误差加大）。所以你得反着走——把 \(w\) 往小了调。

    第 3 步：更新参数（optimizer.step()）— 往反方向挪一步

    \[w_{new} = w_{old} - \eta \cdot \frac{dL}{dw}\]

    设学习率 \(\eta = 0.01\)：

    \[w_{new} = 3 - 0.01 \times 6 = 3 - 0.06 = 2.94\]

    \(w\) 减小了 0.06。

    第 4 步：重复——误差越来越小

    新参数 \(w = 2.94\)，再算一遍 \(L\)：

    \[L = 2.94^2 = 8.6436\]

    误差从 9 降到了 8.64！接着算新梯度 \(2 \times 2.94 = 5.88\)，再减 \(0.01 \times 5.88\)，\(w\) 继续往 0 靠近……

    重复几千次后，\(w \approx 0\)，\(L \approx 0\)。模型就这样从一个”瞎蒙”的状态，一步一步蹭到了最优解。

    成百上千万的参数，干的都是同一件事——各自朝着”让 Loss 变小”的方向，每次蠕动一丁点。

    Q11：为什么叫”梯度下降”？瞎子下山比喻¶

    刚才的 \(L = w^2\) 是一条简单的抛物线，计算机一眼能看全貌。但真实网络有千万个参数，Loss 的图形复杂到根本无法想象——计算机连”长什么样”都不知道，更不知道谷底在哪。

    那它怎么知道往哪走是下坡？

    瞎子下山法：把你蒙上眼睛，丢在一座大山里。要走到谷底，你怎么做？

    你虽然看不见全景，但只要蹲下来摸一下脚底的地面，就能感觉出脚下是往左歪还是往右歪：

    • 脚底往右翘 → 右边高、左边低 → 往左走一步
    • 脚底往左翘 → 左边高、右边低 → 往右走一步

    求导就是”摸脚底”——只告诉你当前这一个点的倾斜方向，不告诉你整座山的全貌。

    为什么算出正梯度要减、负梯度要加？

    规定：往右走 = \(w\) 增加，往左走 = \(w\) 减少。

    • 梯度 > 0（正数）：脚下是 / 形斜坡，右边高、左边低。往右走 = 上山（误差变大），所以得往左：\(w_{new} = w_{old} - 正数\)。
    • 梯度 < 0（负数）：脚下是 \ 形斜坡，左边高、右边低。往右走 = 下山，所以得往右：\(w_{new} = w_{old} - 负数 = w_{old} + |梯度|\)。

    公式 \(w_{new} = w_{old} - \eta \cdot \nabla\) 已经自动处理了方向。

    为什么学习率那么小（0.001 这种级别）？

    瞎子只知道脚底下这一厘米是 / 形状——他完全不知道两步之外是平地还是悬崖。

    大步走 = 赌博：可能一脚跨过谷底到了对面山坡（震荡），甚至直接踩进悬崖（Loss 爆炸）。小步走 = 安全，代价是慢。

    为什么需要成千上万个 Epoch？

    每次只挪微不足道的半步。挪完这半步后，脚底的感觉（梯度）就过期了——必须重新蹲下摸一次（再跑一次 backward()），再挪半步……循环千万次，瞎子终于一步一步蹭到了谷底。

    梯度是一个绝对”短视”的东西——只看当前点的斜率。我们就是靠极其微小的局部感知 + 大量计算资源，硬把参数”蠕动”到最优解的。

    总结速查¶

    问题 一句话答案
    Batch size 影响参数数量？ 不影响，参数只由网络结构决定
    四个相同 Linear 是废工吗？ 不是，初始值不同，训出来各自不同
    权重能初始化为 0 吗？ 不能，打不破对称性，所有神经元永远一样
    CNN vs RNN？ CNN 处理独立图片，RNN 处理序列
    RNN 致命缺点？ 梯度消失，记不住远距离信息
    LSTM 怎么治的？ 加法更新 \(C_t\)，梯度可无损远传
    backward vs step？ backward 只是算斜率，step 才真正改参数
    激活函数为什么要非线性？ 没有它网络就是一条直线，什么都分不开
    为什么门控用 Sigmoid？ 天然输出 0~1，完美模拟开关信号
    backward 在算什么？ 算 Loss 对每个参数的局部斜率（导数）
    为什么叫梯度下降？ 沿着脚下最陡的方向，瞎子摸黑一步步蹭到谷底
