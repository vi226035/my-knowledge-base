    循环神经网络（RNN）¶

    传统的全连接网络和 CNN 都隐含了一个假设：输入之间是独立的。但在很多场景中，输入是一个序列，前后之间有密切关联。

    举个例子，看这句话："我昨天去超市买了一个__"。填什么？"苹果"？"电视"？光看"一个"判断不了，必须结合前面的"超市"来推理。传统网络做不到这一点，因为它不知道"超市"和"一个"之间的时序关系。

    RNN 的核心思想就一句话：网络会"记住"之前看到的内容，用它来影响当前的判断。

    应用场景¶

    RNN 处理的序列任务主要分四种模式：

    一对一（传统网络）¶

    标准的前馈网络。输入一张图 → 输出一个类别。没有序列概念。

    一对多¶

    一个输入，输出一个序列。例如：图像描述生成——输入一张图片，输出一段文字"一只猫坐在窗台上"。

    多对一¶

    一个序列输入，一个输出。例如：情感分析——输入"这部电影真的太好看了"（词序列），输出"正面"。

    多对多（两种）¶

    • 不实时（Seq2Seq）：读完整个输入序列再生成输出。例如：机器翻译——读完整句英文，再输出整句中文。
    • 实时：每读一个输入就立刻输出一个结果。例如：视频逐帧标注——每一帧画面都要判断里面有什么。

    网络结构¶

    RNN 的结构非常简洁——在普通全连接网络的基础上，多加了一条"回头路"：当前时刻隐藏层的输出，会作为下一时刻隐藏层的额外输入。

    \[\begin{aligned} h_t &= f(W_{xh} x_t + W_{hh} h_{t-1} + b_h) \\ y_t &= g(W_{hy} h_t + b_y) \end{aligned}\]

    其中： - \(x_t\) — \(t\) 时刻的输入 - \(h_t\) — \(t\) 时刻的隐藏状态（RNN 的"记忆"） - \(h_{t-1}\) — 上一时刻的隐藏状态（这就是"循环"的来源！） - \(y_t\) — \(t\) 时刻的输出 - \(W_{xh}\) — 输入到隐藏层的权重 - \(W_{hh}\) — 隐藏层到隐藏层的权重（核心：负责传递历史信息） - \(W_{hy}\) — 隐藏层到输出的权重

    关键：\(W_{xh}\)、\(W_{hh}\)、\(W_{hy}\) 在所有时间步共享——同一套参数处理整个序列。

    最简单 RNN 的工作方式¶

    以一个最简 RNN 为例，输入 \(x_t\) 和上一个隐藏状态 \(h_{t-1}\) 送入隐藏层，输出 \(y_t\) 和新的隐藏状态 \(h_t\)：

    时刻 t 的流程：
  x_t + h_{t-1} → [隐藏层] → h_t → [输出层] → y_t
                                    ↓
                            传递给时刻 t+1

    字符级语言模型示例¶

    假设我们要训练一个 RNN 来预测下一个字符。输入序列是 "hello"，词汇表是 {h, e, l, o}：

    训练过程： - 输入 h，期望输出 e（h 后面大概率是 e） - 输入 e，期望输出 l（e 后面大概率是 l） - 输入 l，期望输出 l（第一个 l 后面可能还是 l） - 输入 l，期望输出 o（第二个 l 后可能是 o）

    每个时刻，RNN 输出的是整个词汇表的概率分布。以输入 l 第二个为例，模型综合了前文 "hel" 的信息： - 输出层给出每个字母的概率：h: 0.03, e: 0.05, l: 0.28, o: 0.64 - 选概率最大的 o 作为预测

    注意：\(h_t\) 是当前时刻的隐藏状态（综合了从 \(t=0\) 到现在的所有信息），\(y_t\) 是从 \(h_t\) 推出的输出。

    正向传播¶

    RNN 的正向传播就是在每个时间步重复同样的计算。

    第 t 步的完整计算：

    1. 输入 \(x_t\) 和上一时刻的隐藏状态 \(h_{t-1}\)
    2. 计算当前隐藏状态：\(h_t = \tanh(W_{xh} x_t + W_{hh} h_{t-1} + b_h)\)
    3. 计算当前输出：\(y_t = \text{softmax}(W_{hy} h_t + b_y)\)（分类任务）

    然后 \(h_t\) 传给 \(t+1\) 步，循环往复。

    直观理解¶

    就像一个学生听课记笔记： - \(h_t\) = 当前大脑里的理解（结合刚听到的 \(x_t\) + 之前已理解的内容 \(h_{t-1}\)） - \(y_t\) = 基于当前理解给出的回答 - 每听到一个新词，都更新一次"理解"和"回答"

    反向传播（BPTT）¶

    RNN 的反向传播叫 BPTT（Backpropagation Through Time，通过时间的反向传播）——本质还是链式法则，只是需要沿着时间轴往回传梯度。

    为什么叫"通过时间"？¶

    普通网络只有"层"的维度，RNN 多了"时间"的维度。误差从最后一个时刻开始，沿着时间线一步步往前传，计算每个时刻 \(h_t\) 对最终损失贡献了多少。

    了解这大概流程即可：复合函数 + 链式法则沿时间展开。编程时框架会自动处理。

    展开的计算图¶

    把 RNN 按时间展开后就像一个很深的网络——长度为 T 的序列，展开后就是 T 层的"伪深度网络"。每一层共享同样的参数，但梯度要穿过的路径很长，这就埋下了梯度消失的隐患。

    截断 BPTT¶

    序列很长时（如一段几百词的文本），梯度从头传到尾代价太大，而且序列开头的词和末尾的词关联可能很小。实际做法：分块计算——把长序列切成长度固定的小块（如 100 步），在每块内做完整的 BPTT，块之间断开梯度流。

    这样做既节省计算，也避免了不相关的远距离依赖干扰当前训练。

    多层 RNN¶

    就像 CNN 可以堆叠多层，RNN 也可以——把第一层 RNN 的输出作为第二层 RNN 的输入：

    输入 → RNN层1 → RNN层2 → RNN层3 → 输出

    每一层的每个时间步都有自己独立的隐藏状态。多层 RNN 能学到不同时间尺度的特征——浅层抓短期模式，深层抓长期模式。但通常 2~3 层就足够，再多反而容易过拟合。

    双向 RNN¶

    普通 RNN 的问题是：\(h_t\) 只包含过去的信息（\(h_0\) 到 \(h_{t-1}\)），看不到未来。

    但很多任务需要上下文。比如："我的手机__了"，填"坏"还是"没电"取决于后面的语境。正向 RNN 看到"手机"后只能猜，不够准。

    双向结构¶

    双向 RNN 同时跑两条 RNN：

    • 前向 RNN：从左到右读序列，\(h_t^{\rightarrow}\) 包含过去信息
    • 后向 RNN：从右到左读序列，\(h_t^{\leftarrow}\) 包含未来信息
    • 最终输出：\(h_t = [h_t^{\rightarrow}; h_t^{\leftarrow}]\)（拼接）

    这样就可以同时利用前后文信息了。代价：无法实时处理——必须读完整个序列才能跑后向 RNN。

    RNN 的核心问题：梯度消失¶

    记得 RNN 的 \(W_{hh}\) 在每个时间步都会被乘一次。经过 \(T\) 步后，梯度中会出现 \(W_{hh}^T\) 项：

    • 如果 \(|W_{hh}| < 1\)：连续乘 \(T\) 次后梯度过近于 0（梯度消失）→ 远距离的依赖关系学不到
    • 如果 \(|W_{hh}| > 1\)：连续乘 \(T\) 次后梯度爆炸 → 参数更新剧烈震荡

    梯度消失是 RNN 最致命的问题——理论上 RNN 能处理任意长度的依赖，但实际只能记住约 10~20 步内的内容，再远就"忘了"。这也是 LSTM 和 GRU 被发明的原因。
