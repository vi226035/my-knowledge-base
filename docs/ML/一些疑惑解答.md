# 深度学习常见疑惑解答

把入门时最容易卡住的问题，用大白话一次性讲清楚。

---

## 第一部分：神经网络的底层逻辑

### Q1：batch size 变大，模型参数会变多吗？

**不会。** 参数数量只由网络结构决定，和每次喂多少数据无关。

**打个比方**：神经网络是一台榨汁机（固定的刀片 + 电机）。一次丢 1 个苹果还是一个筐 64 个苹果，榨汁机本身没有任何变化。变化的是"一次处理多少"而已。

**数学上**：`[64, 28] × [28, 128] = [64, 128]`。64 张图同时乘的是**同一份** 128 维的权重矩阵——矩阵乘法的并行特性让 batch 内所有样本共享参数。

---

### Q2：代码里写了四个一模一样的 `nn.Linear(28,128)`，为什么不是重复废工？

**因为它们是四个独立的"盲盒"——公式一样，但肚子里装的随机初始值完全不同。**

每次调用 `nn.Linear()`，PyTorch 在内存里开辟一块新地盘，塞入**完全不同的随机数**作为初始权重。经过训练后，四个盲盒各自学到不同的东西——一个变成遗忘门，一个变成输出门，各司其职。

> 关键不是公式长什么样，而是**权重值被训成了什么样**。

---

### Q3：初始权重为什么不能全设为 0？

**必须打破对称性。全 0 = 所有神经元永远做一模一样的事。**

**打个比方**：请了 128 个专家，但初始时全都白纸一张（权重为 0）。你扔一张猫图进去，算出来的误差对 128 个专家的"批评"完全一样（梯度相同）→ 他们每次接收同样的修正 → 训到最后 128 个专家全部一模一样——等于只请了一个人，白花了 127 份工资。

**给微小的随机初始值**，让专家们起跑线不同，最终各自掌管不同的特征。

## 第二部分：CNN 和 RNN 的分工

### Q4：CNN 能处理图像，为什么还要 RNN？

**CNN 是一次性快照，RNN 是连续剧。**

| | CNN | RNN |
|---|-----|-----|
| 适合数据 | 独立图片 | 序列（文字、语音、股价） |
| 输入之间 | 互不相关 | 前后依赖 |
| 核心机制 | 卷积核滑动 | 隐藏状态 $h_t$ 沿时间传递 |

RNN 每读一个词，就把理解（隐藏状态 $h_t$）传给下一个时刻——像击鼓传花一样，当前的理解建立在之前所有内容之上。

## 第三部分：从 RNN 到 LSTM

### Q5：RNN 有什么致命缺点？

**长期失忆——梯度消失。**

RNN 每一时刻的记忆传递都要做矩阵连乘。如果乘数小于 1，连乘几十次后梯度趋近于 0——第一页的内容彻底"糊了"。

**打个比方**：复印一份文件，每次复印是前一次的 90% 清晰度。复印 50 次后，第一页的字已经完全看不清了。RNN 实际只能记住约 10~20 步内的信息。

---

### Q6：LSTM 怎么解决失忆？（核心）

**LSTM 造了一条"加法高速公路"——细胞状态 $C_t$。**

RNN 的信息更新是**全量覆盖**：$h_t = \tanh(W x_t + U h_{t-1})$

LSTM 的信息更新是**选择性加法**：$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$

代入具体符号：
- $C_{t-1}$：旧的长期记忆
- $f_t$：遗忘门（哪些旧记忆要扔掉，0~1 之间的系数）
- $i_t$：输入门（哪些新信息要写入）
- $\tilde{C}_t$：候选记忆（本次想写入的新内容）
- $\odot$：逐元素相乘

因为是**加法**（+）而不是矩阵连乘（×），梯度可以沿着 $C_t$ 无损地传回几十步甚至上百步之前。这就是 LSTM 能记住"远距离依赖关系"的根本原因。

---

### Q7：LSTM 复杂的公式到底在做什么？

用**记课堂笔记**来比喻 LSTM 每个组件：

| 组件 | 符号 | 课堂比喻 | 作用 |
|------|------|---------|------|
| 细胞状态 | $C_t$ | 主笔记本 | 记录所有重要考点（长期记忆） |
| 隐藏状态 | $h_t$ | 临时脑子 | 当前这秒想着的事，传给下一时刻 |
| 遗忘门 | $f_t$ | 橡皮擦 | 把笔记本上过时的旧内容擦掉 |
| 候选状态 | $\tilde{C}_t$ | 草稿本 | 根据新内容总结出的"本节课精华" |
| 输入门 | $i_t$ | 荧光笔 | 判断草稿上的内容有多少值得写入笔记本 |
| 输出门 | $o_t$ | 聚光灯 | 从笔记本里挑出当前时刻最需要的内容输出 |

四步流程一目了然：

1. **擦旧**：$f_t$ 擦掉笔记本上该忘的东西 → $f_t \odot C_{t-1}$
2. **写新**：$i_t$ 判断草稿本 $\tilde{C}_t$ 有多少值得写 → $i_t \odot \tilde{C}_t$
3. **更新**：旧的 + 新的 = 当前笔记本 → $C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$
4. **输出**：$o_t$ 聚光灯照笔记本，提取当前需要的部分 → $h_t = o_t \odot \tanh(C_t)$

## 第四部分：训练四步曲

### Q8：`loss → zero_grad → backward → step` 到底在干什么？

这是模型从瞎蒙进化到精准的全部秘密。用**厨师做菜**来理解：

```python
loss = criterion(outputs, labels)  # 第1步：尝菜，给差评
optimizer.zero_grad()              # 第2步：撕掉昨天的记过本
loss.backward()                    # 第3步：反向追责，精准定锅
optimizer.step()                   # 第4步：执行处分，修改配方
```

**第 1 步 `loss = ...` — 顾客尝菜，给出差评**

算网络的瞎猜答案和正确答案差多少。得分越高 = 越难吃 = 错得越离谱。

**第 2 步 `optimizer.zero_grad()` — 先撕掉昨天的记过本**

PyTorch 的梯度默认**累加**！不清空的话这批的失误会和上批的混在一起。所以必须先清零，只算当前这批的账。

**第 3 步 `loss.backward()` — 反向追责，精准定锅**

系统顺着流水线往回倒推（链式求导），算出每个参数"多调 1 单位能让差评减多少"——这叫**梯度**。

比如算出来"糖的参数多 0.1 能大大降低差评"，就给糖的参数标一个负梯度（说明该加糖了）；"盐的参数多 0.1 让差评暴增"，就给盐标一个正梯度（说明盐放多了）。

**注意**：执行完这行时，参数本身**还没被修改**——只是拿到了修改建议书。

**第 4 步 `optimizer.step()` — 执行处分，修改配方**

真正修改权重的时刻。厨师拿着建议书（梯度）× 学习率（比如 0.001），正式更新每个参数：

$$W_{new} = W_{old} - \eta \cdot \frac{\partial L}{\partial W}$$

其中 $\eta$ 是学习率，$\frac{\partial L}{\partial W}$ 是梯度。

一次改动极小，但几万个回合下来，瞎猜变成神准。

## 总结速查

| 问题 | 一句话答案 |
|------|-----------|
| Batch size 影响参数数量？ | 不影响，参数只由网络结构决定 |
| 四个相同 Linear 是废工吗？ | 不是，初始值不同，训出来各自不同 |
| 权重能初始化为 0 吗？ | 不能，会打破不了对称性 |
| CNN vs RNN？ | CNN 处理独立图片，RNN 处理序列 |
| RNN 致命缺点？ | 梯度消失，记不住远距离信息 |
| LSTM 怎么治的？ | 加法更新 $C_t$，梯度可以无损远传 |
| backward vs step？ | backward 只是算建议，step 才真正改参数 |
