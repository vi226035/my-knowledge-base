# 梯度下降优化算法

训练神经网络的目标就一件事：**找到让 Loss 最小的那组参数**。但怎么找？从一个随机点出发，每步朝下山的方向挪一点。

所有优化器的本质都是：

$$w_{new} = w_{old} - \eta \cdot \text{（某种修正过的方向）}$$

区别在于"方向"怎么算——有些原样照搬梯度，有些加惯性，有些每个参数单独调步长。这些"怎么算"的差异，直接决定了训练是几十秒收敛还是几小时还在原地踏步。

---

## 速查：6 种优化器一句话

| 优化器 | 一句话 | 什么时候用 |
| ------------ | -------------------------- | -------------- |
| **SGD** | 纯梯度下降，最原始 | 调试、教学 |
| **Momentum** | 加惯性，滚下山更快 | CNN 训练（配合 SGD） |
| **NAG** | 先探路再迈步，比 Momentum 聪明 | 同上，更稳定 |
| **AdaGrad** | 每个参数单独配学习率 | 稀疏特征（NLP 词频） |
| **RMSprop** | 修了 AdaGrad 越走越慢的 bug | RNN / LSTM 训练 |
| **Adam** | Momentum + RMSprop 合体，开箱即用 | **默认首选**，适用最广 |

---

## 1. SGD（随机梯度下降）— 原始且诚实

### 公式

$$w_{t+1} = w_t - \eta \cdot \nabla L(w_t)$$

符号说明：
- $w_t$：当前参数（第 $t$ 步时）
- $\eta$：学习率（每一步迈多大脚）
- $\nabla L(w_t)$：损失函数对 $w$ 的梯度（脚底当下的坡度）

### 大白话

每一步沿着最陡的方向往下走。不拐弯、不加惯性、不预设——完完全全听当前脚下坡度的。

### 打个比方

你闭着眼睛站在山坡上，每走一步都蹲下来摸一下脚底感受坡度，然后朝脚底最歪的方向迈一步。走到哪算哪。

### 训练过程详解

SGD 每次随机取一个 mini-batch 的数据算梯度。因为是随机抽的，这个梯度不是全局最准确的方向，只是一个"大概对"的方向。噪音很大，但噪音也是好事——噪音足够大时，能帮你从鞍点附近"抖"出去。

**为什么叫随机**？因为每次只随机取一小部分数据（比如 64 张图）算梯度，而不是在所有数据上算精确梯度。

### 参数 $\eta$（学习率）是怎么影响训练的？

$\eta$ 是 SGD 唯一的超参数，但它极其敏感：

| $\eta$ 太大 | 一步跨过谷底，出现震荡甚至发散，Loss 忽大忽小 |
| $\eta$ 太小 | 乌龟爬，收敛极慢，可能训到世界末日 Loss 还没到底 |
| $\eta$ 刚好 | 平滑下降，Loss 曲线稳定往下走 |

### 适用场景

- 教学演示（代码最少，最容易理解什么叫"梯度下降"）
- 作为 baseline 对比其他优化器的效果
- 配合学习率衰减做最终精调（一些论文追求极致精度时用）

### 不适用场景

- RNN/LSTM：梯度在时间轴上变化剧烈，固定 $\eta$ 完全应付不了
- 稀疏数据：每个特征出现频率不同，一个全局学习率不公平
- 深度网络：浅层梯度小、深层梯度大，一个 $\eta$ 管不了所有层

### 优点

- 简单到极致，一行代码
- 收敛到最优解有理论保证（对于凸函数）
- 噪音本身是一种隐式正则化，有时泛化更好

### 缺点

- 一步一停，走不快
- 遇到峡谷地形（一个方向极陡、另一个方向极平），会在陡壁上反复震荡——就像乒乓球在两面墙之间弹来弹去，沿峡谷底部的方向反而走不动
- 学习率要精细调，调大了震荡、调小了太慢
- 容易被困在鞍点（高维空间中鞍点远比局部极小值常见）：四周都是平的，梯度 = 0，SGD 原地不动

## 2. Momentum（动量法）— 给 SGD 装上惯性轮

### 公式

$$\begin{aligned}
v_t &= \beta v_{t-1} + \eta \cdot \nabla L(w_t) \\
w_{t+1} &= w_t - v_t
\end{aligned}$$

符号说明：
- $v_t$：当前速度（累积的动量），单位与梯度相同
- $v_{t-1}$：上一步的速度
- $\beta$：动量系数（0~1 之间），$\beta$ 越接近 1，惯性越大、旧速度衰减越慢

### 大白话

不光看当前脚底的坡度，还带着上一步的速度惯性。像推一个铁球下山——如果连续几步都往一个方向，速度会越叠越快；如果梯度左右摇摆，惯性会把两边的摆动互相抵消掉。

### 打个比方

SGD 是每步停住重新摸地再走。Momentum 是你推着一个有惯性的大铁球下山——它自己越滚越快，小的坑洼石子直接碾过去，不会被小障碍拦住。

### $\beta$ 是怎么影响训练的？

$\beta$ 决定了"惯性记忆"有多长。以 $\beta=0.9$ 为例，过去 10 步的梯度各有权重（最近的占 0.1，上一步 0.09，再上步 0.081...），而 30 步以前的梯度贡献已趋近于 0。

| $\beta=0$ | 退化为 SGD，没有惯性 |
| $\beta=0.5$ | 只记近 2~3 步，惯性很弱 |
| $\beta=0.9$ | 经典值，记近 10 步，工业界标准 |
| $\beta=0.99$ | 惯性接近 100 步，下坡极快但谷底可能刹不住 |
| $\beta$ 太大 | 冲向谷底时速度太快刹不住车，反复冲过头（overshooting） |
| $\beta$ 太小 | 接近 SGD，震荡去不掉 |

### 为什么能抑制震荡？

震荡方向上的符号不断反转（两次正两次负），累加速度时会互相抵消。而真正下山的方向符号一致，会持续叠加加速。这就是为什么 Momentum 在峡谷地形能比 SGD 快好几倍——它把震荡方向的步给抵消了，有效步只朝下山方向走。

### 适用场景

- CNN 图像分类（配合 SGD，工业界经典组合）
- 中等到大 batch size 的训练
- 需要快速穿过鞍点区域的任务

### 不适用场景

- 梯度本身已经很大的情况——叠上惯性直接冲飞
- 参数数量极少的小模型——Momentum 的优势体现不出来

### 优点

- 加速收敛，尤其梯度方向一致时（长下坡）
- 自然抑制震荡——梯度左右摇摆时，惯性自动平滑掉锯齿
- 更容易冲过鞍点（惯性直接碾过去）

### 缺点

- 多了一个超参数 $\beta$
- 惯性太大可能冲过头，谷底附近反复震荡
- 还是全局统一的学习率，不同参数没法用不同步长

## 3. NAG（Nesterov 加速梯度）— 先探路再迈步

### 公式

$$\begin{aligned}
v_t &= \beta v_{t-1} + \eta \cdot \nabla L(w_t - \beta v_{t-1}) \\
w_{t+1} &= w_t - v_t
\end{aligned}$$

和 Momentum 的唯一区别：梯度不是在当前位置 $w_t$ 算的，而是在"按惯性预判下一步会到的位置" $w_t - \beta v_{t-1}$ 算的。

### 大白话

Momentum 的问题——它蒙着眼睛往下冲，冲过头了才知道回头。NAG 在冲之前先往前伸手探一下路："前面如果是坑，我就提前拐。"

### 打个比方

Momentum 是你闭着眼睛推铁球，到哪算哪。NAG 是你一边推一边睁半只眼往前瞟——看到前面有弯道提前减速调方向。

### 为什么"探头位置"的梯度比当前位置的好？

当你在谷底附近时，当前位置的梯度几乎是 0（谷底是平的），但探头位置已经能看到"再走就要上坡了"的警告信号——此时当前位置梯度 ≈ 0 不会触发减速，而探头位置的梯度会提前往回拉，避免冲过头。

这是 Momentum 和 NAG 最关键的差异：Momentum 是在原地看到悬崖才开始刹车（可能来不及），NAG 是提前看到悬崖就减速绕开了。

### 参数作用同 Momentum

$\beta$ 控制惯性大小，典型值 0.9。

### 适用场景

- 需要更稳定收敛时替代 Momentum
- CNN 训练（精度敏感任务中优于 Momentum）
- 不希望训练后期因惯性震荡的任务

### 优点

- 比 Momentum 收敛更稳定，振荡更小
- 在谷底/最优点附近会自动提前减速
- 收敛曲线更平滑，超调（overshoot）更少

### 缺点

- 实现略微复杂——要算两次（先跑步、再算梯度）
- 还是需要手动调 $\eta$ 和 $\beta$
- 仍然是全局学习率，不同参数没有区别对待

## 4. AdaGrad（自适应梯度）— 每个参数单独配学习率

### 公式

$$\begin{aligned}
G_t &= G_{t-1} + (\nabla L(w_t))^2 \\
w_{t+1} &= w_t - \frac{\eta}{\sqrt{G_t + \epsilon}} \cdot \nabla L(w_t)
\end{aligned}$$

符号说明：
- $G_t$：所有历史梯度平方的累加和（每个参数一个独立的 $G_t$）
- $\epsilon$：极小常数（如 $10^{-8}$），防止除以零
- $G_t$ 越来越大 → 分母越来越大 → 该参数的有效步长越来越小

### 大白话

SGD/Momentum/NAG 的问题是：所有参数共用同一个学习率。但实际网络里，有些参数（如高频词的 embedding）每步都在更新，梯度很大；有些参数（如低频词）几乎不动。给它们同样的步长不公平。

AdaGrad 的做法：给**每个参数**单独记录它历史上总共更新了多大。该参数历史上越活跃 → 有效步长缩小越多 → 防止它"学过头"。不活跃的参数 → 分母小 → 保持大步长继续探索。

### 打个比方

每个参数都有一张"履历表"。频繁被大梯度更新的参数（如高频词"的"）——已经见多识广了，给它小步微调；很少被更新的参数（低频词"貔貅"）——还没想明白呢，给它大步激进学习。

### 为什么对稀疏特征特别友好？

在 NLP 任务中，"的"这个字每句话都出现，如果学习率相同，"的"的词向量很快被各种梯度来回拉扯、最后烂掉。而"貔貅"只出现了几次，学习率相同的话几乎训不到。

AdaGrad 自动给"的"很小步长、给"貔貅"很大步长——低频特征得到充分训练，这是 SGD 做不到的。

### 核心致命缺点：学习率单调递减

$G_t$ 不断累加正数（平方），从不减少。分母 $\sqrt{G_t}$ 随着训练越来越大，有效学习率越来越小。训练时间长了，所有参数的有效步长都趋近于 0——**网络还没收敛就先"冻死了"**。这就是为什么 AdaGrad 不适合深度网络长时间训练。

### 适用场景

- 稀疏特征突出（NLP 词袋模型、推荐系统的 user/item embedding）
- 浅层网络（训不久就停了，不会冻死）
- 数据分布极不均衡（长尾分布）时

### 不适用场景

- 深度网络（50+ 层）——没训完学习率就 0 了
- 长时间训练——后期完全学不动

### 优点

- 每个参数自适应学习率，不需要手动精细调
- 对稀疏特征极度友好，低频特征训练充分
- 超参数少（只有初始 $\eta$）

### 缺点

- 学习率单调递减到 0，后期网络完全"冻住"
- 不适合深度网络
- 不适合非凸优化（会被困在不好的局部极小值）

## 5. RMSprop — 解决 AdaGrad"越走越慢"的 bug

### 公式

$$\begin{aligned}
G_t &= \beta G_{t-1} + (1-\beta)(\nabla L(w_t))^2 \\
w_{t+1} &= w_t - \frac{\eta}{\sqrt{G_t + \epsilon}} \cdot \nabla L(w_t)
\end{aligned}$$

$\beta$ 通常设 0.9。

### 大白话

和 AdaGrad 唯一区别：$G_t$ 不再是累加所有历史梯度平方，而是用**指数移动平均**——新的梯度权重 $1-\beta=0.1$，旧历史以 $\beta=0.9$ 的比率逐次衰减。

效果：最近 10 步左右的梯度大小决定当前步长，几十步以前的旧梯度影响逐渐归零。学习率不会无限趋零，而是随梯度变化动态调整。

### 打个比方

AdaGrad 是把从小到大的每一笔收支都记录在案，200 年前的旧账也要拿出来当证据限制今天的花销。RMSprop 只保留最近一段时间的账单——旧的过期作废，始终保持对"当前状态"的合理评估。

### $\beta$ 的影响

| $\beta=0.9$（默认） | 窗口约 10 步，平滑适中 |
| $\beta=0.99$ | 窗口约 100 步，极度平滑但响应慢——梯度突然变化时步长调整迟钝 |
| $\beta=0.5$ | 窗口约 2 步，跟随快但毛刺多——步长忽大忽小 |

### 为什么是 RNN/LSTM 训练的经典选择？

RNN 的梯度在不同时间步差异极大——刚开头几步梯度很小，越往后梯度越大。如果用固定学习率，后期参数会被巨大梯度假摔死，前期又走不动。RMSprop 自动给大梯度时刻用小步长、给小梯度时刻用大步长，完美适配"梯度幅度变化剧烈"的场景。

### 适用场景

- **RNN / LSTM** 训练（经典首选，至今仍很多 paper 在用）
- 非平稳目标函数（在线学习、强化学习）
- 梯度幅度在不同时刻/不同层差异很大的情况

### 不适用场景

- 追求极简（多了一个 $\beta$，不如 Adam 全能）

### 优点

- 解决了 AdaGrad 学习率衰减到 0 的致命问题
- 每个参数自适应步长，梯度剧烈变化时稳得住
- RNN/LSTM 训练久经考验

### 缺点

- 多了一个衰减参数 $\beta$ 需要设定
- 没有动量（虽然没有大碍，但收敛速度不如 Adam）
- 不如 Adam 开箱即用（$\beta$ 设错了会影响效果）

## 6. Adam（自适应矩估计）— 当下的绝对王者 ⭐

### 公式

$$\begin{aligned}
m_t &= \beta_1 m_{t-1} + (1-\beta_1) \nabla L(w_t) &&\text{（一阶矩估计，类似 Momentum 的惯性方向）} \\
v_t &= \beta_2 v_{t-1} + (1-\beta_2) (\nabla L(w_t))^2 &&\text{（二阶矩估计，类似 RMSprop 的自适步长）} \\
\hat{m}_t &= \frac{m_t}{1-\beta_1^t} &&\text{（$m_t$ 的偏差校正）} \\
\hat{v}_t &= \frac{v_t}{1-\beta_2^t} &&\text{（$v_t$ 的偏差校正）} \\
w_{t+1} &= w_t - \frac{\eta \cdot \hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}
\end{aligned}$$

**默认值**：$\eta=0.001$，$\beta_1=0.9$，$\beta_2=0.999$，$\epsilon=10^{-8}$

### 大白话

把 **Momentum 的惯性**（$m_t$）和 **RMSprop 的自适应步长**（$v_t$）焊在一起。再加一个**偏差校正**——因为 $m_t$ 和 $v_t$ 初始都是 0，训练前几步会被"偏压"得比真实值小，除以 $1-\beta_1^t$ 就是放大补偿。训了几步之后 $1-\beta_1^t$ 趋近 1，校正自然消失。四个模块各司其职：

| 一阶矩 $m_t$ | 记住方向惯性，平滑震荡（Momentum 贡献） |
| 二阶矩 $v_t$ | 给每个参数调独立步长（RMSprop 贡献） |
| $\hat{m}_t,\,\hat{v}_t$ | 偏差校正，防训练初期 $m_t/v_t$ 被初始 0 压扁 |
| 最终分子分母 | 方向用 $m_t$，步长用 $1/\sqrt{v_t}$，两全其美 |

### 打个比方

Adam 是一个全能司机：有预判力（Momentum——知道该往哪冲），会看路踩油门（RMSprop——平坦路猛踩、颠簸路轻踩），而且刚启动时不会因为不知道车况就猛打方向盘（偏差校正——温柔起步）。

### 四个超参数怎么调？

**$\eta$（学习率）**：Adam 的默认 0.001 对 90% 的任务都 OK。如果 Loss 震荡得厉害 → 降至 0.0001。如果 Loss 下降太慢 → 试 0.003 或 0.01。这是最值得调的参数。

**$\beta_1$（一阶矩衰减）**：控制"惯性"有多大。默认 0.9。如果训练曲线毛刺太多 → 提至 0.95（更强的平滑）。如果模型被过去的梯度影响太久、反应迟钝 → 降至 0.8。

**$\beta_2$（二阶矩衰减）**：控制"步长自适应"有多平滑。默认 0.999 意味着过去约 1000 步的梯度平方都在影响当前步长。如果训练后期不稳定 → 降至 0.99 或 0.9（只关心近期梯度变化，灵活调步长）。

**$\epsilon$（防止除零）**：基本不用动。除非你在做半精度（float16）训练，可以改大到 $10^{-4}$ 防溢出。

### 适用场景

- **所有场景**。CNN、RNN、LSTM、Transformer、GAN、RL 几乎任何神经网络架构
- 新手首选（默认参数直接跑，不用调）
- 快速实验和原型验证

### 不适用场景（极少数）

- 追求极致泛化精度时，精心调参的 SGD+Momentum + 学习率衰减可能略胜（论文有过报道，但差距通常很小）
- 超大 batch（>4096）时可能不稳定
- 极深 Transformer 大模型（BERT/GPT 规模）推荐用 AdamW

### 优点

- **开箱即用**：默认超参数几乎不需要调
- 每个参数自适应步长，各走各的节奏
- 有惯性、不易困鞍点
- 目前工业界和学术界的绝对标准

### 缺点

- 泛化质量有时不如精心调参的 SGD+Momentum
- 极深 Transformer 需要 AdamW 等变体
- 四个超参数（虽然默认值够用了）
- 内存比 SGD 大（要存 $m_t$ 和 $v_t$，每个参数配两个缓存，内存 ≈ 3× 纯 SGD）

|--------|:----:|:----:|:----:|:----:|:----:|---------|
| SGD | ✗ | ✗ | 多 | 慢 | 基准 | 教学、baseline |
| Momentum | ✗ | ✓ | 中 | 较快 | 2x SGD | CNN 图像任务 |
| NAG | ✗ | ✓ | 中 | 较快 | 2x SGD | CNN，替代 Momentum |
| AdaGrad | ✓ | ✗ | 少 | 前期快后期停 | 2x SGD | 稀疏数据 |
| RMSprop | ✓ | ✗ | 少 | 快 | 2x SGD | RNN/LSTM |
| **Adam** | **✓** | **✓** | **极少** | **快** | **3x SGD** | **通用首选** |

## 实际怎么选？场景速查

| 你是什么情况 | 用什么 |
|------------|--------|
| 刚入门学深度学习 | Adam，默认参数不要动 |
| 训练 CNN 分类图片 | SGD + Momentum（老派）或 Adam（省事） |
| 训练 RNN / LSTM 文本/序列 | Adam 或 RMSprop |
| 词嵌入训练（NLP） | AdaGrad 或 Adam |
| Transformer 大模型（BERT/GPT） | AdamW |
| GAN 训练 | Adam（$\beta_1=0.5$ 降惯性，$\beta_2=0.999$） |
| 强化学习 | RMSprop 或 Adam |
| 强实时嵌入式推理 | SGD + Momentum（内存最小） |
| 精度竞赛，追求那 0.1% | SGD + Momentum + Cosine 衰减 + 网格搜参 |
| 不知道该用哪个 | Adam，不解释 |

> **一句话金句**：先用 Adam 跑通。不满意效果，再换 SGD + Momentum + 学习率衰减精调。

## 超参数速查

| 优化器 | 参数 | 推荐值 | 怎么调 |
|--------|------|--------|--------|
| SGD | $\eta$ | 0.01 ~ 0.1 | 震荡就降，太慢就加 |
| Momentum | $\eta$ | 0.01 | 同上 |
| | $\beta$ | 0.9 | 震荡多就提至 0.95 |
| NAG | $\eta$ | 0.01 | 同 Momentum |
| | $\beta$ | 0.9 | 同 Momentum |
| AdaGrad | $\eta$ | 0.01 | 初始值，会自衰减 |
| RMSprop | $\eta$ | 0.001 | 调法与 SGD 类似 |
| | $\beta$ | 0.9 | 梯度变化快就提至 0.99 |
| **Adam** | $\eta$ | **0.001** | **最值得调的参数** |
| | $\beta_1$ | 0.9 | 一般不动，train 曲线上蹿下跳才提 |
| | $\beta_2$ | 0.999 | 后期不稳定才降（0.99 或 0.9） |
| | $\epsilon$ | $10^{-8}$ | 基本不动 |

## AdamW 和 Adam 有何区别？（附注）

AdamW 是修复版的 Adam。原始 Adam 把 L2 正则化（权重衰减）放在了梯度里一起算，这导致自适应学习率和正则化互相干扰——步长大的参数正则化弱，步长小的参数正则化强。

AdamW 把权重衰减**直接加在参数上**，与梯度解耦：

$$w_{t+1} = w_t - \eta \cdot \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon} - \eta \lambda w_t$$

最后一项 $\eta \lambda w_t$ 就是解耦后的权重衰减。效果比 Adam + L2 好，是目前 Transformer 训练的标准配置。
