# 知识库总结规范

本文件记录论文总结文档的格式规范，后续所有总结任务均按此标准执行。随时按你的要求修改。

---

## 1. 章节组织

- 一章一个 `.md` 文件，不合并
- 文件名用简短中文描述，去掉"第X章"前缀
  - 例：`雷达数字信号中频接收.md`
- 每章开头加来源备注：
  ```
  > 本章整理自《论文全名》第X章
  ```

## 2. 导航配置

`mkdocs.yml` 的 `nav` 用全中文简短标题：
```yaml
nav:
  - 信号处理:
      - 雷达数字信号中频接收: 信号处理/雷达数字信号中频接收.md
```

## 3. 图片规范

- 图片源来自 `knowledge_base_material/` 下对应论文的 MinerU 解析目录
- 通过 junction `docs/kb_material/` 引用源文件，**不复制图片**
- 图片路径格式：`../kb_material/论文目录名/images/xxx.jpg`
- 每张图下方紧跟图注：
  ```
  ![](../kb_material/.../xxx.jpg)

  **图2.1：描述文字**
  ```
- 图注加粗居中（CSS 自动处理）
- 图片居中、统一最大宽度 90%、圆角阴影（CSS 自动处理）

## 4. 公式规范

- 使用 `$$...$$` 块级公式 + `pymdownx.arithmatex` 扩展
- 渲染引擎：KaTeX（`katex.min.js`，jsdelivr CDN）
- **每个公式后必须附符号说明**：
  ```
  $$
  f_s \ge 2f_H
  $$

  其中：
  - $f_s$：采样频率
  - $f_H$：信号最高频率
  ```
- 复杂公式集中写一个 `#### 符号说明` 小节

## 5. MkDocs 配置要点

```yaml
# mkdocs.yml 关键配置
theme:
  name: material
  language: zh

markdown_extensions:
  - pymdownx.arithmatex:
      generic: true

extra_javascript:
  - https://cdn.jsdelivr.net/npm/katex@0.16.11/dist/katex.min.js
  - javascripts/katex.js

extra_css:
  - https://cdn.jsdelivr.net/npm/katex@0.16.11/dist/katex.min.css
  - stylesheets/extra.css
```

## 6. 资源目录结构

```
docs/
├── javascripts/
│   └── katex.js          # KaTeX 渲染脚本
├── stylesheets/
│   └── extra.css         # 自定义样式
├── kb_material/          # → junction 到 knowledge_base_material/
├── 信号处理/
│   └── 各章.md
└── ...
```

## 7. 样式要点（extra.css）

- 图片：`display:block; margin:auto; max-width:90%; border-radius; box-shadow`
- 公式：`overflow-x:auto; margin:0.8em 0`
- 表格：居中、圆角、表头灰底
- 引用块：左侧蓝色边框
- 二级标题：上间距、底部分割线
